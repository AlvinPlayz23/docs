# Project Management

PROJECT-X is a community effort! The core team of contributors aims at organizing the project in a way that allows you to understand where the current areas of focus are.

## Getting Involved

If you want to contribute to PROJECT-X, there are several ways to get started:

1. **Check the High Priority Items** - Look at the current high priority items in the [About](/about) page
2. **Submit Pull Requests** - Contribute code improvements or new features
3. **Report Issues** - Help identify bugs or suggest improvements
4. **Improve Documentation** - Help make the documentation more comprehensive

## Development Process

The PROJECT-X development process is designed to be open and collaborative:

<Steps>
  <Step title="Fork the Repository">
    Start by forking the [PROJECT-X repository](https://github.com/bolt-new/project-x) to your GitHub account.
  </Step>
  <Step title="Set Up Your Environment">
    Follow the instructions in the Quickstart guide to set up your local development environment.
  </Step>
  <Step title="Choose a Task">
    Look at the current issues or high-priority items and choose something to work on.
  </Step>
  <Step title="Create a Branch">
    Create a new branch for your feature or bug fix with a descriptive name.
  </Step>
  <Step title="Submit a Pull Request">
    Once your changes are ready, submit a pull request with a clear description of what you've done.
  </Step>
</Steps>

## Community Guidelines

When contributing to PROJECT-X, please follow these guidelines:

- Be respectful and inclusive in all communications
- Write clear commit messages and documentation
- Follow the existing code style and patterns
- Test your changes thoroughly before submitting
- Provide context and reasoning for your changes
